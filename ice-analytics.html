The Challenge

As we scaled features, data became fragmented across ad networks, IAP stores, and gameplay event logs. Product managers couldn't answer simple questions like "Did the new shop layout increase ARPDAU?" without asking an engineer to pull raw logs.

Goal: Build a self-serve dashboard that democratizes data access and standardizes how we measure success.

The Solution

ICE Analytics is a centralized dashboard (Internal/Concept) that aggregates data into three core views:

1. Executive Health Check

High-level DAU/MAU, Retention (D1, D7, D30), and Revenue pulses. Used for daily standups.

2. Economy Sink/Source Balance

Visualizes currency flow. Detects inflation early by comparing "Earn per User" vs "Spend per User" per level.

3. Feature Funnels

Step-by-step conversion tracking for specific features (e.g., Tutorial Completion, Shop Purchase Flow).

Key Metrics Tracked

  • ARPDAU (Average Revenue Per Daily Active User): The north star for monetization health.
  • LTV (Lifetime Value): Projected value based on D7/D30 cohorts.
  • Stickiness (DAU/MAU): Measuring habit formation.
  • Conversion Rate: % of users who make a first purchase.

Product Learnings

Building this tool taught me that data without context is noise. The initial version had too many charts; the V2 focused only on actionable metrics (Red/Green indicators) which increased adoption by the design team significantly.
